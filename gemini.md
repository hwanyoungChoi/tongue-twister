## 4. 상세 게임 메커니즘 (Game Logic)

### 공통 설정

- **인원**: 2~6명.
- **구성**: 총 3라운드 (라운드당 10장, 총 30장 카드 소진).
- **벌칙**: 시작 전 직접 입력 또는 리스트 선택 가능, 최종 탈락자에게 자동 적용.

### 모드 1: ⏱ 타이머 모드 (경쟁형)

- **로직**: 카드당 제한 시간(ex. 3초) 내 읽기 -> '통과' 버튼 클릭.
- **점수**: 남은 시간이 점수로 환산됨 (n초 남음 = +n점).
- **패널티**: 시간 초과 시 목숨 -1 (기본 2개 제공), 0개 시 탈락.
- **특수 카드**:
  - **보너스**: 시간 추가/멈춤, 턴 건너뛰기, 목숨 추가.
  - **벌칙**: 시간 감소, 카드 흔들림/블러, 두 장 읽기, 방해 요소(먼지/눈뽕) 등장.

### 모드 2: ⏳ 양심 모드 (자율형)

- **로직**: 시간 제한 없이 읽은 후 본인이 '성공/실패' 직접 선택.
- **패널티**: 실패 시 1벌점, 누적 2회 시 탈락.
- **특수 카드**: 턴 건너뛰기, 목숨 추가, 카드 흔들림, 틀렸던 카드 재등장.

## 5. 개발 및 답변 지침

- **답변 스타일**: 모든 설명은 핵심을 먼저 말하는 **두괄식**으로 작성할 것.
- **컴포넌트**:
  - `components/ui`: 기능 위주의 순수 컴포넌트 (스타일 최소화).
  - `components/common`: 프로젝트 디자인 명세가 반영된 확장 컴포넌트 (GameButton, GameSwitch 등).
- **스택**: React + Vite 기반 (CSR), Tailwind CSS v4 스타일링 적용. 단일 페이지(Single Page Application) 구조로 개발하며, 게임 컴포넌트 내의 모든 화면 전환은 `react-router-dom`을 사용하여 처리함.
