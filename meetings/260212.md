## 260212 기술 검토 및 질문거리 정리

### 기술 스택

- 프레임워크 및 UI/CSS: React(Vite) Tailwind CSS를 활용. Vite의 빠른 HMR을 통해 고퀄리티 UI 디테일 및 애니메이션 작업 효율을 극대화하기 위해 선정.

- 배포: Vercel을 통해 정적 호스팅 및 배포를 진행.

- 애니메이션 (Lottie): Lottie 에셋 제공 시 즉시 적용.

- 연출 (Framer Motion): 0.99x 호버 스케일, 페이지 전환, 모달 노출/닫기, 카드 넘김 등 앱과 같은 부드러운 인터랙션 구현 및 게임 연출에 활용.

### 진행 계획

- 페이지 구조: 모든 게임 흐름을 **상태 기반으로 관리하는 단일 페이지 애플리케이션(SPA)**으로 개발.

- 데이터 관리: Figma 내 문장을 기준으로 하드코딩 진행. (향후 확장성을 고려한 데이터 구조 설계 병행)

- 최적화: 스플래시 화면 노출 동안 **폰트, 사운드, Lottie 등 핵심 에셋을 프리로딩(Preloading)**하여 진입 시 성능 최적화.

### 질문 리스트

#### 현재 개발 관련

- 최대 너비 설정: PC 환경도 고려하여 웹사이트의 최대 너비(max-width)를 설정해야 할까요? 아니면 모바일 타겟으로만 보고 별도의 최대 너비 설정 없이 개발을 진행해도 괜찮을까요?

- 단위 선택: 다양한 기기 화면 크기에 대응하기 위해 고정 px 단위를 사용할지, 화면 너비에 비례하여 크기가 조절되는 rem 단위를 활용할지?

#### 나중에 논의가 필요한 질문

- 효과음 적용 범위: 효과음은 어떤 요소들에 적용하는 것이 좋을까요?
  - 모든 버튼에 공통으로 적용할지
  - 특정 버튼에만 적용할지
  - 특정 시점이나 특정 영역에만 적용할지

- 안드로이드 앱 대응: 향후 안드로이드 앱 출시를 고려한다면, 현재 웹 버전을 웹뷰로 띄우는 형태로 갈지, 아니면 ReactNative와 같은 웹 기술 기반으로 AOS 앱을 개발하는 방안도 고려

- 공유하기 방식: 서버/DB 구축 없이 URL 파라미터에 게임 결과 데이터를 인코딩하여 전달하는 방식(서버리스)으로 진행해도 괜찮을지?
